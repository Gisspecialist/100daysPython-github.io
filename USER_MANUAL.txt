Coding Notebook Sheets App — Upgraded User Manual
=================================================

Version: Terminal Input Runner + Embedded Help Upgrade

1. Overview
-----------
This application is a browser-based Python coding notebook inspired by the 100 Days of Python sheet layout. It lets you create coding sheets by day, keep multiple sheets under the same day, save all notebook data in one JSON file, and run Python code in a terminal-style runner.

2. Main Upgrades in This Version
--------------------------------
- Embedded Help / User Guide panel below the Support My WGU Journey section.
- Searchable help topics.
- Collapsible help sections.
- Expand All and Collapse All buttons.
- Guided Walkthrough mode.
- Live runnable examples that load directly into the Python Runner.
- Terminal-style input handling: input() questions are answered in the Output panel, not popups.

3. Days and Sheets
------------------
Use the Select Day dropdown to choose the day you want to work on. Use Add Day to create a new day. Use New Sheet for Selected Day to create multiple sheets under the same day. Previous Sheet and Next Sheet move between sheets for the currently selected day.

4. Sheet Fields
---------------
Each sheet includes:
- Concept
- Day
- Filename
- Code lines 01–18
- Comments
- Summary
- Lecture

5. Running Python
-----------------
Click Load 01–18 into Runner to move the sheet code into the runner input panel. Click Run Python to execute the code.

The runner prints an online-style transcript:
- Run started
- Initializing environment
- Installing packages
- Running code
- Your Python output
- Code completed successfully or error details

6. Answering input() Questions
------------------------------
When your code uses input(), the prompt appears in the Output area. Type your answer directly beside the prompt and press Enter. This replaces popup prompts and behaves more like an online Python terminal.

7. Installing Packages
----------------------
Enter package names in the Install package box, separated by commas, then click Install with micropip. Packages require the Pyodide runtime and internet access.

8. JSON Storage
---------------
Autosave stores your work in the browser. Export Full Notebook JSON downloads one file containing all days and all sheets. Import JSON restores a previous notebook.

9. Embedded Help / User Guide
-----------------------------
The Help / User Guide panel is below the Support My WGU Journey container. It includes:
- Overview and getting started
- Days, multiple sheets, and navigation
- Saving and JSON storage
- Python runner and terminal input
- Live examples
- Troubleshooting

Use the search box to filter help topics. Use Expand All or Collapse All to manage the panel. Use Start Guided Walkthrough to learn the app step by step.

10. Live Examples
-----------------
The Live Examples section can load sample code into the runner:
- Band Name Generator
- Simple Calculator
- Loop Practice

After loading an example, click Run Python to test it.

11. Best Practices
------------------
- Use one concept per sheet.
- Keep each day's work grouped under that day.
- Use multiple sheets when a lesson has more than one code example.
- Export the full JSON notebook regularly.
- Check the Python output panel for tracebacks when debugging.
